Методы расчета планарных волноводов¶

Перечень API класса multi_layer_slab_solver для расчета планарных волноводов.

class multi_layer_slab_solver.MultiLayerSlab(ns, nc, n_layer, d_layer, wl)¶

MultiLayerSlab - класс, позволяющий находить эффективные показатели преломления планарного волновода с произвольным числом слоев.

Переменные

  • ns (float) – ПП подложки

  • nc (float) – ПП покровного слоя

  • n_layer (numpy.array) – ПП волноведущих слоев

  • d_layer (numpy.array) – Толщины волноведущих слоев

  • wl (float) – Длина волны

Muller(f, x0, x1, x2, eps, Max)¶

Метод Muller - функция для поиска корней алгебраических уравнений. Корнями могут быть как реальные, так и комплексные числа.

Параметры

  • f – функция, соответствующая исследуемому уравнению

  • x0 – пробные значения, относительно которых будет начинаться поиск корней уравнения

  • x1 – пробные значения, относительно которых будет начинаться поиск корней уравнения

  • x2 – пробные значения, относительно которых будет начинаться поиск корней уравнения

  • eps (float) – точность поиска корня

  • Max (int) – число итераций приближения к корню

SlabSolver(eps, Max)¶

Метод SlabSolver - функция по поиску и сортировке всевозможных мод планарного волновода с произвольным числом слоев. Суть метода заключается в поиске всех решений дисперсионных уравнений планарного волновода с произвольным числом слоев, сортировка и запись решений в словари. Так как решений может быть несколько, метод содержит числы, где итерируется по пробным значениям ЭПП в пределах максимального и минимального ПП. Это позволяет найти всевозможные корни.

Параметры

  • eps (float) – точность поиска корня

  • Max (int) – число итераций приближения к корню

Переменные

  • n_max (complex) – значение максимального ПП, используемого в качестве границы для поиска решения

  • n_min (complex) – значение минимального ПП, используемого в качестве границы для поиска решения

  • neff_init_0 (complex) – пробное значения ЭПП, относительно которго ищется решение

  • neff_init_1 (complex) – пробное значения ЭПП, относительно которго ищется решение

  • neff_init_2 (complex) – пробное значения ЭПП, относительно которго ищется решение

  • dneff (complex) – шаг изменения начальных значений ЭПП (в некотором смысле точность поиска решения)

  • mode_control (int) – флаг, обозначающий

Return TE_modes

списки, куда записываются все решения

Return TM_modes

списки, куда записываются все решения

neff_TE(neff_init)¶

Метод neff_TE - функция, представляющая собой дисперсионное уравнение планарного волновода с произвольным числом слоев. На вход метода подается значение эффективного показателя преломления (ЭПП), исследуемого волновода. Когда ЭПП, поступающий на вход метода, соответствует истинному значению ЭПП волновода, то функция выдает значение 0.

Модовые, полевые и другие волноводные значения могут быть комплексными. В методе нет циклов, вместо них применяется массивы numpy. При вычислении квадратных корней от отрицательных с помощью встроенных функций numpy могут появляться ошибки. Numpy не останавливает программу, а показывает ошибку RuntimeError в массиве с отрицательными числами при вычислении корня появляются значения nan. Чтобы иметь возможность вычислять корни от отрицательных числе, в исходном numpy массиве должно быть хотя бы одно комплексное число. Тогда корень от отрицательных чисел вычисляется правильно. В переменных gamma, kappa в массив к каждому числу добавляется произведение 1j * 0, чтобы числа стали комплексными и была возможность вычисления корня.

Параметры

neff_init (complex) – пробное значение ЭПП

Переменные

  • beta (complex) – постоянная распространения

  • NumLayers (int) – число волноводных слоев планарного волновода

  • gamma_subs (complex) – постоянная, характеризующие поле внутри волновода

  • gamma_clad (complex) – постоянная, характеризующие поле внутри волновода

  • kappa (complex) – постоянная, характеризующие поле внутри волновода

Результат

эффективный показатель преломления TE-моды

#### Литература #### Computational photonics. An introduction with Matlab
